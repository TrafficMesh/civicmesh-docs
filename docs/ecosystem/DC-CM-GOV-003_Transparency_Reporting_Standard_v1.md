# 1. Mandatory Annual Report Structure


| Section | Required Content | Data Source |
|---|---|---|
| 1 — Network Overview | Total active nodes by type, node map (published separately), communities in network, year-over-year node count change | Node registry |
| 2 — Incident Volume | Evidence packages generated by violation type, reviewed by officers, citations issued, dismissal rate, false positive rate by violation type | Evidence pipeline metrics |
| 3 — Revenue Distribution | Total enforcement revenue generated, distribution breakdown (municipal share, platform fee, CCSC member patronage), average patronage per node owner, government stipends distributed | Revenue ledger |
| 4 — Law Enforcement Access | Number of federation requests received by legal authority type (WARRANT/REASONABLE_GROUNDS/EXIGENT/ROUTINE), number approved, number denied, average notification time, communities affected | Federation audit log |
| 5 — Privacy Performance | Privacy complaints received and resolved, data subject access requests fulfilled, data deletion requests fulfilled, breach events (if any) and response actions | Privacy complaint register |
| 6 — AI Performance | Model version deployed, false positive rate by violation type, dismissal rate trend, model retraining events, accuracy benchmark results | Model performance metrics |
| 7 — Cooperative Governance | AGM attendance, board elections, resolutions passed, extraordinary motions | CCSC governance records |
| 8 — Corrective Actions | Any audit findings from prior year and status of corrective actions | Audit report |


# 2. Publication Requirements


| Timing | Published 30 days before CCSC Annual General Meeting. Distributed to all CCSC members. Published on civicmesh.ca/transparency/{community-id}/{year}. |
|---|---|
| Format | PDF for member distribution. Machine-readable JSON version for public data consumers. Manager Dashboard generates both automatically from audit log data. |
| Mandatory Review | Community manager reviews auto-generated draft before publication. Manager DID signature on approved report is logged. |
| Anti-Redaction Policy | No data may be redacted from the annual transparency report. All fields must be populated. If a field is zero, it is reported as zero — not omitted. Omission is equivalent to hiding information. |
