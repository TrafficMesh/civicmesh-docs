# Design Brief


| Data-forward design | Insurance executives are quantitative. Every slide should have a number. No conceptual slides without supporting data. |
|---|---|
| The telematics problem visual | Slide 2: a simple split-screen. Left: 'What telematics dongles capture' (driver behaviour during monitored period). Right: 'What CivicMesh captures' (verified behaviour continuously, third-party validated). The difference should be visually obvious. |
| Tier comparison | A clear product tier table — Tier 1 aggregate / Tier 2 individual / Tier 3 incident — with use cases, data included, and pricing. Actuaries will screenshot this slide. |
| The premium discount flywheel | One circular diagram: premium discount → opt-in → better data → accurate pricing → larger discount → more opt-in. Simple. Visual. Memorable. |
| Privacy compliance | One slide, short. 'PIPEDA compliant. Consent-gated. No adverse use. Annual audit. Canadian data residency.' Five bullet points. No paragraphs. |
