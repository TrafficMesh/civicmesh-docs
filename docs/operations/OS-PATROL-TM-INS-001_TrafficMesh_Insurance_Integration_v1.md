# 1. TrafficMesh-Specific Insurance Data Products


| Product | Data Unique to TrafficMesh | Insurance Application |
|---|---|---|
| Corridor risk score | Violation density by type + time-of-day + weather condition + road condition from digital twin — for every road segment in Ottawa | Actuarial pricing by postal code + route pattern. More granular than any existing data source. |
| UBI driving behaviour record | Lane discipline (from node footage), following distance, time-of-day profile, construction zone behaviour, school zone compliance — for opted-in TrafficMesh node-owning drivers (dashcam opt-in) | Individual underwriting at renewal. Premium reduction for clean 12-month record. More trusted than self-reported telematics. |
| Collision corridor analysis | Near-miss detection from node footage (vehicles braking hard in proximity) + actual collision events from digital twin change detection | Identify high-risk corridors for pricing. Claim frequency prediction by postal code. |
| Seasonal risk adjustment | Winter road condition index + violation frequency change in winter vs summer — correlated by corridor | Seasonal premium adjustment data. Currently estimated by insurers from weather data alone — TrafficMesh provides ground truth. |


# 2. UBI Backbone Technical Design


| Opt-In Mechanics | TrafficMesh node-owning drivers who opt in through the Member App consent to: (1) their dashcam footage being analysed for driving behaviour, and (2) the resulting behaviour summary being shared with their selected insurer at renewal. |
|---|---|
| Behaviour Extraction | From opted-in driver's own dashcam footage: following distance estimation (distance to vehicle ahead from camera + speed from OBD-II), lane change events, hard braking events (OBD-II acceleration data), construction zone speed compliance, school zone speed compliance. |
| Privacy Constraint | Behaviour summary does not include specific route data — insurer receives statistical summaries (e.g. '87th percentile following distance, 3 hard braking events per 1,000km') not location traces. |
| Third-Party Validation | The defining advantage: this data is not self-reported by the insured driver. It is generated by the enforcement infrastructure — the same infrastructure that generates citations. Insurers cannot dispute the data quality. |
